Elixir Phoenix Development

Techracers Elixir.

Being working with every emerging technology, we at techracers didn't took too long to try our hands with Elixir. Once started with it, we found that, there is no other efficient/clean language available to write concurrent program that can utilize all the available cores & memory efficiently. It being a functional language and having immutable data, really help us avoid pitfalls which are faced while writing concurrent/multithreaded programs.

Elixir being derived from the Erlang, it inherits a lot of features like:- distributed, concurrent, fault tolerant, hot code swap.

Phoenix is cool, too!

We at techracers sees Phoenix as a successor to Rails. Phoenix provides everything we need to develop a web application. Built on top of Elixir, it is highly scalable. Influenced by Rails, it has conventions and follows MVC pattern. if Requirement is real-time app, it provides Phoenix Channels to implement real-time features.

When to choose Elixir over Ruby

Rails is a full featured advanced framework for developing web applications. Still, Rails is not a good fit for cases where we need to build highly scalable real-time application. Problem lies within the language(Ruby) itself on top of which the it is built. Ruby is not concurrent by itself and it's really hard to make Ruby programs utilize all the available resources efficiently. If a highly scalable, future proof & real-time application is required, the best choice is Phoenix!
